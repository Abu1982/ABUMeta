# 安全策略

## 支持范围

当前安全维护重点是 ABUMeta 开源演示版主分支。

## 不要公开提交的内容

- API Key
- 账号密码
- Cookie / Session
- 本地数据库
- 外部站点抓取结果
- 日志与报告产物
- `data/action_journal.jsonl`
- `data/reports/*.md`
- `data/reports/*.json`
- `data/reports/*.jsonl`
- `data/logs/*.log`

## 安全报告方式

如果你发现安全问题，请不要直接公开提交可利用细节。

建议在私下渠道报告以下信息：

- 问题描述
- 影响范围
- 复现步骤
- 风险等级判断
- 可能的修复建议

## 当前已知高风险区域

- 外部抓取能力
- 外部 LLM 服务接入
- 本地文件、行动账本与日志输出
- 可选动态浏览器后端

## 处理原则

- 优先修复高风险泄露问题
- 优先阻止密钥、抓取数据和运行资产被误提交
- 第三方依赖升级时同步检查许可证和安全影响
